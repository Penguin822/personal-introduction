🎯 Architecture Overview

在「靜香鞋類進銷存系統」中，我負責端到端系統開發。此專案最大的亮點在於前期嚴謹的系統分析與架構設計，而非單純程式實作。良好的地基是維持系統擴充性與穩健度的關鍵。

🧠 System Analysis

為了解決零售業中會員、銷貨、庫存與財務四大模組的複雜邏輯，我從需求塑模開始，繪製兩階資料流程圖（DFD），完整定義資料流動與系統行為。

🗂 Database Normalization

在資料庫設計中，建立完整 ERD 並進行正規化處理，確保資料的一致性與查詢效能：

  • 解耦與獨立：將會員資訊從訂單表拆分，避免資料冗餘。
  • 關聯性約束：使用 PK / FK 建立嚴謹關聯，透過「會員編號」串接資料。
  • 防呆機制：從資料庫層級確保訂單與帳務資料一致性，降低後端邏輯錯誤風險。
⚙️ Implementation
  • 使用 Lazarus IDE 快速開發 20+ UI 畫面，實踐快速迭代。
  • 專注後端 CRUD 操作與邊界條件的防呆邏輯。
  • 完成進銷存模組與財務自動化系統的整合。
