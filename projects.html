學生憂鬱症成因分析與預測模型

  • 專案目標： Kaggle 競賽數據探勘專案。使用 Python (Scikit-learn) 建構高穩定性的預測模型。
▲ Kaggle 競賽數據集
1. 特徵工程：量化影響力
  • 核心策略： 運用 Information Gain 和 Gain Ratio 評估屬性。在龐雜數據中精準篩選「訊號」，排除雜訊。
▲ Information Gain 與 Gain Ratio 演算法特徵評估結果
2. 模型評估與效能對標
  • 混淆矩陣分析： 預測錯誤代價不同 (偽陰性 vs 偽陽性)。不盲從 Accuracy，從混淆矩陣掌握模型犯錯細節。
  • ROC 曲線比較： 確保模型具備實質區分訊號的能力。
  • 交叉驗證 (Cross-Validation)： 嚴格防範過擬合 (Overfitting)，最終選出具備極高穩定性的 XGBoost 模型。

AWS 雲端自動化設備維護與監控系統

  • 專案目標： 展現從「實體基礎設施」到「無伺服器 (Serverless) 架構」的雲端整合能力。
▲ AWS 系統整體架構圖
1. 基礎設施部署與資安控管
  • 部署 Ubuntu EC2 (網頁伺服器) 與 RDS (資料庫)。
  • 嚴格控管 RDS 的 Security Group，僅放行特定 IP 連線，確保資料安全。
▲ EC2 伺服器部署與 RDS 安全群組設定
2. 無伺服器自動化排程監控
  • 導入 AWS EventBridge 取代傳統排程器。
  • 時間觸發時，啟動 Lambda 函數 掃描 RDS 設備期限。
  • 若期限將至，透過 SNS 主動發送 Email/簡訊告警，達成全自動化維運。
▲ 運用 EventBridge 觸發 Lambda 進行 Serverless 自動化排程
